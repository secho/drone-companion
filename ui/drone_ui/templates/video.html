How to view this stream (H.264)

QGroundControl has a built-in H.264 viewer. Application Settings → Video:

  • Source: UDP h.264 Video Stream
  • UDP Port: {{ video.gcs_port }}
  • Disable When Disarmed: unchecked (so video shows even before you arm the FC)

Switch to the Fly view — video appears in the lower-left PiP within ~2 s.

If your QGC is the official v5 macOS DMG, the Source dropdown only shows "Video Stream Disabled" — that build was compiled without GStreamer. Either downgrade to QGC v4.4.x, build v5 from source, or use the standalone viewer below.

Standalone H.264 viewer (run on your GCS — works regardless of QGC):

gst-launch-1.0 -q udpsrc port={{ video.gcs_port }} caps="application/x-rtp,media=video,clock-rate=90000,encoding-name=H264,payload=96" \
  ! rtpjitterbuffer latency=80 \
  ! rtph264depay ! avdec_h264 \
  ! videoconvert ! autovideosink sync=false

Prereqs: brew install gstreamer gst-plugins-base gst-plugins-good gst-libav on macOS, or sudo apt install gstreamer1.0-tools gstreamer1.0-plugins-good gstreamer1.0-libav on Debian.

How to view this stream (MJPEG)

QGroundControl's built-in viewer doesn't speak MJPEG-over-RTP, so use a standalone GStreamer pipeline. The trade-off versus H.264: MJPEG uses 3–5× the bandwidth, but each frame is independent — packet loss only damages a single frame instead of cascading green-block corruption across many. Best for lossy wireless links.

One-line viewer command (run on your GCS):

gst-launch-1.0 -q udpsrc port={{ video.gcs_port }} caps="application/x-rtp,media=video,clock-rate=90000,encoding-name=JPEG,payload=26" \
  ! rtpjitterbuffer latency=80 drop-on-latency=true \
  ! rtpjpegdepay ! jpegdec \
  ! videoconvert ! autovideosink sync=false

Or, if you have the drone-companion repo cloned on your GCS:

scripts/gcs-view-mjpeg.sh {{ video.gcs_port }}

Prereqs: brew install gstreamer gst-plugins-base gst-plugins-good on macOS, or sudo apt install gstreamer1.0-tools gstreamer1.0-plugins-good on Debian. No gst-libav needed for MJPEG.
